# 2주차 과제

## 배열 (Array)

데이터들을 메모리 공간에 연속적으로 저장하는 자료구조
메모리 상에서 연속적으로 저장되어 있기 때문에, index를 이용한 접근이 용이하다.
배열의 크기는 처음 생성할 때 고정되며 변경이 불가능하다.

### 장점
인덱스를 통한 빠른 접근이 가능하다.

### 단점
삽입과 삭제가 복잡하고 오래 걸린다.
크기가 고정되어 있다.

## 단일 연결 리스트 (Linked List)

여러 개의 노드들이 순차적으로 연결된 형태를 갖는 자료구조
첫번째 노드를 head, 마지막 노드를 tail이라고 한다.
각 노드는 데이터와 다음 노드를 가리키는 포인터로 이루어져 있다. (단일 연결 리스트)
배열과 다르게 메모리를 연속적으로 사용하지 않는다.
언제든지 크기 변경이 가능하다.
순차적인 접근에는 불리할 수 있으나, 삽입과 삭제 등이 용이하다.

### 장점
삽입과 삭제 용이하다.
크기 변경이 자유롭다.

### 단점
임의 접근이 불가능하여, 처음부터 순차적으로 탐색을 진행해야 한다.
